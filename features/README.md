# Acceptance / BDD

Сюда помещаются пользовательские и интеграционные сценарии.

Когда использовать:

- изменение затрагивает форму;
- нужен проверяемый пользовательский flow;
- unit seam слабый или вторичен;
- важно наблюдаемое поведение, а не внутренняя реализация.

Пример стартового сценария лежит в `features/examples/tdd-starter.feature`.
